# 用 Spike + 原生参照做编译器差分模糊测试

## 核心思路

编译一个"是某宿主语言子集"的语言时，**用宿主编译器作为黄金参照**：同一份源文件，一边
用我们的编译器生成目标汇编、在模拟器上跑，一边用宿主 `cc` 原生编译运行，**比对程序退出码**
（取低 8 位）。源语言是 C 子集，这一招几乎零成本地给出"正确答案"。

## 在 macOS 上跑 RV32：用 Spike 裸机 + HTIF

macOS 没有 qemu 用户态。用 Spike（功能级 ISA 模拟器）以裸机方式跑：

- 写一段最小启动代码：设好 `sp`、`call main`、把 `(exit_code<<1)|1` 写入 HTIF 的
  `tohost`。Spike 据此退出，**其进程退出码即程序返回值**。
- 链接脚本把代码放在 `0x80000000`，并提供 `tohost/fromhost` 符号。
- 启动代码与链接脚本**只用于本地测试**，绝不混进编译器的标准输出。

注意：`riscv64-unknown-elf-gcc -march=rv32im -mabi=ilp32` 即可生成 RV32；即便其 `cc1`
因依赖缺失不可用，`as`/`ld` 通常仍可独立做汇编与链接。

## 模糊器要点

- **生成"无 UB"的程序**，否则参照与被测可能各自做不同的"未定义"行为而误报。
  实践上有两条互补策略：
  1. 按构造限制幅度（叶子取小常数、深度受限、除数非零），杜绝溢出与除零；
  2. 利用"两边都用 `-O0`、都是 32 位补码、有符号溢出在实践中都回绕"——此时即便溢出，
     两边结果一致，差分仍然有效（但别让宿主在编译期对 UB 常量做激进折叠）。
- **覆盖不同形态**：纯表达式（练优先级/结合/短路）、带界限循环（练控制流/分支布局）、
  多函数非递归调用（练调用约定/寄存器分配/溢出）。递归只调用更早定义的函数以保证终止。
- **给模拟器加超时**：被测若把程序错编成死循环，必须超时计为失败，否则测试卡死。
- **双后端互验**：朴素后端与优化后端互为参照，能在没有原生参照时也发现问题。

## 价值

这套体系在本项目里实测暴露并定位了：C 运算符优先级、全局声明的 token 二次消费、
形参冲突漏建等多个缺陷。**几千个随机程序 + 全部手写用例**通过，是放心做激进优化的底气。
