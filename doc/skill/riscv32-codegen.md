# RV32 代码生成要点

## 调用约定（ILP32）

- 参数：`a0–a7`，超出 8 个走栈（调用者放在 `sp+0, sp+4, …`，被调用者在其入口
  `sp` 加上自身帧大小处读取）。
- 返回值：`a0`。
- 调用者保存：`t0–t6`、`a0–a7`、`ra`；被调用者保存：`s0–s11`、`sp`。
- 栈 16 字节对齐。

## 栈帧布局（一个稳妥的版本）

`sp` 在低地址，自下而上：

```
[传出参数区][溢出槽][被保存的 s 寄存器][ra]
```

- 仅在**非叶函数**保存 `ra`；仅保存**实际用到**的 `s` 寄存器。
- 帧偏移可能超过 12 位立即数范围（`lw/sw/addi` 的 imm 是 12 位有符号）：写一个
  统一的访存助手，偏移越界时用一个临时寄存器构造地址，保证任意规模函数都正确。

## 一个简化寄存器分配的实用技巧：把 `a0–a7` 排除出分配池

调用点和函数入口最棘手的是"并行传送/寄存器洗牌"：若实参的源寄存器和目标 `a` 寄存器
互相占用，朴素地逐个 `mv` 会互相破坏。

**做法**：让 `a0–a7` 只用于传参/返回，不参与通用分配。于是：
- 调用点 `materialize(arg_i, a_i)` 的**源都是 `t/s/立即数/溢出槽`、目标都是 `a`**，
  两个集合不相交，任意顺序传送都安全；
- 函数入口把 `a_i` 搬进各形参的 `t/s` 寄存器，同理无冲突。

代价是少了 8 个可分配寄存器，但对小程序几乎无损，换来实现上的简洁与正确。`a` 寄存器
在非传参时段还可当额外 scratch。

## 有符号"除/模 2 的幂"的正确序列

C 语义向零取整，而 `srai` 向负无穷取整，需先加修正偏置：

```
# x / 2^k :  偏置 = (x<0)?(2^k-1):0
srai t, x, 31          # 全 1 或全 0
srli t, t, 32-k        # 取低 k 位 = 2^k-1 或 0   (k=1 时可直接 srli x,31)
add  t, x, t
srai d, t, k
# x % 2^k :  d = x - (x/2^k)*2^k
... 上述得到商 q ...
slli t, q, k
sub  d, x, t
```

## 指令选择小账

- 立即数型优先：`addi/slti/seqz/snez`、与 0 比较用 `zero` 寄存器。
- 比较-分支融合：比较结果只喂给紧随的分支时，直接 `blt/bge/beq/bne`，别物化 0/1。
- 块布局直落：让分支的一个后继成为顺序下一块，省掉无谓的 `j`。
- 叶函数不动 `ra`、不建多余帧。
